# Security Policy

## Supported Versions

We only provide security updates for the latest version of `asyncflowstate`. If you encounter a security issue, please upgrade to the latest version as soon as possible.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of `asyncflowstate` seriously. If you believe you have found a security vulnerability, please report it to us by following these steps:

1. **Do not** open a public issue.
2. Email your report to [INSERT SECURITY EMAIL or use GitHub Advisories].
3. Include as much detail as possible, including:
   - Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting)
   - Product version
   - Steps to reproduce
   - Potential impact
   - Any suggested fix or mitigation

We will acknowledge your report within 48 hours and provide a timeline for a fix within 7 days. Once a fix is verified, we will coordinate a public disclosure with you.

## Disclosure Process

1. Vulnerability is reported privately.
2. We confirm the vulnerability and determine the impact.
3. We work on a fix.
4. A new version is released.
5. The vulnerability is publicly disclosed (e.g., via a GitHub Security Advisory).

Thank you for helping us keep `asyncflowstate` secure!
